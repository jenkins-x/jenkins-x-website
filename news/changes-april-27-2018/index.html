Changes for April 27 2018

Now Skaffold, DevPods, Anchore, MiniShift and much more…

April 27, 2018

Changes for April 27 2018

Its 16 days since our last blog post and we’ve been really busy again!

Here’s some of the main highlights:

  • we have moved to using skaffold for building docker images so that we can easily take advantage of things like Google Container Builder or kaniko
  • new DevPods are a thing; making it easy to use your kubernetes cluster for building, testing and running apps
  • new integration with Anchore for security scanning via jx get cve or in a pipeline you can use jx step post build - we’ll blog separately about that next week
  • thanks to the work of @LinuxSuRen we now have the initial Chinese translation of the Jenkins X documentation
  • we now support MiniShift as well as AKS, AWS, GKE and vanilla kubernetes clusters.
  • we now have a much more flexible mechanism of handling Secrets/Credentials so dealing with other git providers is now much simpler
  • we finally got lots more architecture documentation on how Jenkins X fits together.

The Jenkins X Futures page has been updated to give a better feel for where we are.

Also if you are going to be at KubeCon then please come to the session by James Strachan on Jenkins X or pop by the CloudBees both and say hi!
